负载均衡轮询算法：rest接口第几次请求数 % 服务器集群总数量 = 实际调用服务器位置下标，每次服务启动后rest接口技术从1开始。
    List<ServiceInstance> instances = discoveryClient.getInstances("CLOUD-PARYMENT-SERVICE");
    如：
        List[0] instances = 127.0.0.1:8003
        List[1] instances = 127.0.0.1:8002
        List[2] instances = 127.0.0.1:8001
    8001 + 8002 + 8003 组合成为集群，它们共计2台机器，集群总数为2，按照轮询算法原理：
    当总请求数为1时：1 % 3 = 对应下标位置为1，则获得服务地址为127.0.0.1:8002
    当总请求数为2时：2 % 3 = 对应下标位置为2，则获得服务地址为127.0.0.1:8001
    当总请求数为3时：3 % 3 = 对应下标位置为0，则获得服务地址为127.0.0.1:8003
    当总请求数为4时：4 % 3 = 对应下标位置为1，则获得服务地址为127.0.0.1:8002
    当总请求数为5时：5 % 3 = 对应下标位置为2，则获得服务地址为127.0.0.1:8001
    当总请求数为6时：6 % 3 = 对应下标位置为0，则获得服务地址为127.0.0.1:8003
    如此类推......
